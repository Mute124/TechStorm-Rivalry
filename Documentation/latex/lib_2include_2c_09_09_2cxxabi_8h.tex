Demangling routine. ABI-\/mandated entry point in the C++ runtime library for demangling. 


\begin{DoxyParams}{Parameters}
{\em \+\_\+\+\_\+mangled\+\_\+name} & A NUL-\/terminated character string containing the name to be demangled.\\
\hline
{\em \+\_\+\+\_\+output\+\_\+buffer} & A region of memory, allocated with malloc, of {\itshape \texorpdfstring{$\ast$}{*}\+\_\+\+\_\+length} bytes, into which the demangled name is stored. If {\itshape \+\_\+\+\_\+output\+\_\+buffer} is not long enough, it is expanded using realloc. {\itshape \+\_\+\+\_\+output\+\_\+buffer} may instead be NULL; in that case, the demangled name is placed in a region of memory allocated with malloc.\\
\hline
{\em \+\_\+\+\_\+length} & If {\itshape \+\_\+\+\_\+length} is non-\/null, the length of the buffer containing the demangled name is placed in {\itshape \texorpdfstring{$\ast$}{*}\+\_\+\+\_\+length}.\\
\hline
{\em \+\_\+\+\_\+status} & If {\itshape \+\_\+\+\_\+status} is non-\/null, {\itshape \texorpdfstring{$\ast$}{*}\+\_\+\+\_\+status} is set to one of the following values\+: 0\+: The demangling operation succeeded. -\/1\+: A memory allocation failure occurred. -\/2\+: {\itshape mangled\+\_\+name} is not a valid name under the C++ ABI mangling rules. -\/3\+: One of the arguments is invalid.\\
\hline
\end{DoxyParams}
\begin{DoxyReturn}{Returns}
A pointer to the start of the NUL-\/terminated demangled name, or NULL if the demangling fails. The caller is responsible for deallocating this memory using {\ttfamily free}.
\end{DoxyReturn}
The demangling is performed using the C++ ABI mangling rules, with GNU extensions. For example, this function is used in \+\_\+\+\_\+gnu\+\_\+cxx\+::\+\_\+\+\_\+verbose\+\_\+terminate\+\_\+handler.

See \href{https://gcc.gnu.org/onlinedocs/libstdc++/manual/ext_demangling.html}{\texttt{ https\+://gcc.\+gnu.\+org/onlinedocs/libstdc++/manual/ext\+\_\+demangling.\+html}} for other examples of use.
